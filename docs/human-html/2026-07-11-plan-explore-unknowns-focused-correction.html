Outcome

After this plan lands, a user can enter the unknowns workflow at the phase they are actually in. Pre-build work keeps the existing five-stage walk. Mid-build work records deviations in a task-scoped ledger. Post-build work produces an explainer and a context-sensitive quiz. Every artifact carries a task-derived filename, and every discovery remains visible in one broad map with a plain priority label.

The skill stays focused on unknowns. The broader talk themes about model capability, grief, and ambition do not become runtime rules. The operational source remains A field guide to Claude Fable 5: Finding your unknowns.

Approach

One shared map survives across three phase-aware entrances. A direct mid-build or post-build invocation reuses that map when present and otherwise creates a clearly stamped partial reconstruction, so reconstructed knowledge never looks like a completed pre-build walk.

Locked behavior

  • Scan the repository, Git state, territory profile, memory, and already named sources first. Ask for external context only when relevant context remains missing.
  • Trace callers, consumers, and dependencies until the behavior's origin and impact are explained. There is no arbitrary hop limit, but the trace does not branch into unrelated auditing.
  • Keep all encountered discoveries in the four quadrants. Label each one Changes this task, Nearby finding, or Urgent outside scope.
  • Require a passed quiz for artifact-tier or merge-readiness work. Low-risk inline work has no formal quiz; offer a lightweight understanding check only when meaningful uncertainty remains.
  • Use one task slug across YYYY-MM-DD-unknowns-map-<task>.html, YYYY-MM-DD-implementation-notes-<task>.md, and YYYY-MM-DD-post-build-explainer-<task>.html. Add a numeric suffix for a different task collision.

Risks

  • Reference drift: repeated behavior currently appears in the spine, stage files, and README. Mitigation: change one canonical rule per concern, sweep stale phrases, and add cross-file consistency checks.
  • Map noise: broad capture can bury the decisions that matter. Mitigation: preserve one map but require plain priority labels and sort task-changing findings first.
  • Unbounded investigation: removing a hop limit can turn a walk into an audit. Mitigation: follow only the causal trace needed to explain origin and impact, then stop before unrelated branches.
  • False completeness: a reconstructed map can look like a completed walk. Mitigation: stamp direct-entry fragments with their phase, sources, and skipped stages.
  • Overwriting artifacts: fixed filenames lose prior work. Mitigation: derive a stable task slug, use date-kind-task names, and suffix genuine collisions.

Rollback

The change is documentation and eval data only. If fresh-agent scenarios regress the original three entrances, if the package develops broken references, or if the new wording makes the spine materially larger without improving behavior, revert the focused correction commit while keeping this approved plan as the decision record. No generated user artifacts or external systems need migration.
